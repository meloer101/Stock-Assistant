# Problem Discovery Brief (v1, updated 2026-03-16)

## Problem Statement
- One-liner: 面向个人投资者的“投资研究”场景里，用户需要在统一聊天界面完成轻量股票分析、主题深度研究、以及（研报/年报）PDF 文档问答；但用户对等待时间有硬阈值（轻量 1–3 分钟、深度 3–5 分钟，超时可能放弃），且“可信、可核对”几乎是最高优先级，要求输出包含结论/论点/证据/反例/催化剂并可用链接/标题级输入快速启动。
- Evidence: User statement (2026-03-16)

## Target User
- Primary user: 个人投资者（A 股，自主完成研究与决策）
- Secondary users / stakeholders: 暂无明确组织内角色（待确认：是否存在“带单/社群/内容创作者”等影响者场景）
- Evidence: User statement (2026-03-16)

## User Situation
- Context / Situation: 在统一聊天界面中完成三类任务：轻量股票分析（最高频）、行业/主题深度研究（最高价值）、基于上传 PDF 的文档问答（按需）。
- Trigger:
  - 时间紧迫：看好一只股票、犹豫是否买入，希望快速得到轻量分析结论。
  - 常见触发信号：看到“比较明显的价格波动”（无需系统判断阈值；用户既然提问即已被触发）。
  - 非时间紧迫：想深挖某一主题是否值得建仓，希望做更完整的主题研究。
  - 文档场景：上传研报/年报进行问答核对。
- Goal: 在时间预算内产出结构化结论（结论、核心论点、证据、反例、催化剂），并能被核对以建立/维持信任。
- Constraints:
  - Time budget（强约束）：轻量分析 1–3 分钟；深度研究 3–5 分钟；超过阈值可能放弃。
  - Trust/Verifiability（强约束）：可信和可核对方法几乎最高优先级。
  - Input friction（偏好）：链接/标题即可启动。
  - Document retention（约束澄清）：研报/年报“用完即删”指“不想在界面里留记录”（不等同严格禁止后端留存；仍待确认）。
- Evidence: User statement (2026-03-16)

## Output Contract (current decisions)
- Citation minimum: 仅标题（例如“XX 年报某章节”、“某研报标题”、“某媒体报道标题”）。
- Scenario output ordering: 先情景概率，再证据。
- Evidence weighting: 年报/研报/媒体=4:4:2（用于冲突裁决与综合打分）。
- Evidence: User statement (2026-03-16)

## Job To Be Done (JTBD)
```
When 我作为个人投资者需要在有限时间内对某个标的/主题形成更有把握的判断
I want to 快速完成信息收集→筛选→分析→形成论点/证据链（含引用来源）

```
- Success criteria (user-visible): 等待时间在可接受阈值内；结论附带可点击/可核对的来源；能复用到后续跟踪更新。
- Failure modes: 超时导致放弃；证据不可核对/不可复查导致不信任；关键反例/风险点遗漏导致决策偏差；把事实说反等低级错误导致明显亏损。
- Evidence: User statement (2026-03-16)

## Current Solutions
- What users do today (step-by-step):
  - 轻量买不买：同花顺 → Wind（补充数据/研报/口径）→基于信息做判断。
  - 深度主题：围绕主题在 Wind 上看研报/数据 → 挑龙头（订单/业绩兑现/市占率）→形成建仓判断。
  - 常用验证：对“财务数值/政策条款”优先确认来源是否权威（权威来源则更可信）。
- Tools / workarounds:
  - 平台：同花顺、Wind。
  - 估值口径：PE(TTM)、PE(预测)、PB；对照自身历史与同行。
  - 资金/情绪代理：换手率、龙虎榜。
  - 主题龙头标准：订单、业绩兑现、市占率（硬标准 Top3）。
- Why current solutions persist:
  - 速度与确定性：已有路径能在用户时间阈值内拿到“够用的信息”。
  - 信任与权威：平台/来源的权威性是核心信任锚点。
- Evidence: User statement (2026-03-16)

## Pain Points
- P1: 强时间阈值：轻量 1–3 分钟、深度 3–5 分钟，超时会放弃（即使内容正确也“无效”）。
- P2: 强信任阈值：可信与可核对优先级极高，缺少核对路径会直接削弱可用性。
- P3: 输入摩擦容忍度低：希望“链接/标题即可”，暗示上下文构建成本也是放弃点。
- P4: 最致命错误类型明确：财务数值与政策条款“说反/引用错”会导致明显亏损，容忍度极低。
- P5: 证据冲突需要裁决：用户会对来源做权重（年报/研报/媒体=4:4:2）并在冲突时裁决/在综合时打分。

## Root Causes
- RC1 (mechanism): Because “价格波动触发→需要快速决策（买/不买）”因此用户对 latency 极敏感；任何超过 1–3/3–5 分钟的产出都会被当作失败，从而放弃使用（即“时间超限=价值归零”）。
- RC2 (mechanism): Because 个人投资者面对信息源质量参差（可信 vs 不可信媒体/博主）因此必须能快速核对来源；若核对路径不清晰，会引发不信任/重复核对/重复提问，最终降低可用性。
- RC3 (mechanism): Because 用户偏好“多情景概率（乐观/基准/悲观）+证据”而非单一结论，因此如果分析无法清晰区分“事实/推断/情景假设”，会导致误读与错误行动（尤其是把事实说反的低级错误）。
- RC4 (mechanism): Because 用户的快速核对方式主要依赖“来源权威性”而非逐项复算/逐表核对，因此系统若缺少来源信誉信号与冲突裁决逻辑，会让用户在短时间内无法完成信任判断。
- Non-causes / symptoms: “需要系统直接给结论”不是根因（用户可接受不确定、也不强制结论），核心在于可核对与情景表达的可靠性。
- Evidence: User statement (2026-03-16) + inference (needs validation)

## Opportunity Areas
- OA1: 把“深度研究等待”转成可控：明确时间预算/阶段产出/渐进式返回，提升完成率（不等同于立刻设计方案，先验证其重要性与阈值）。
- OA2: 可核对的引用/来源标注成为默认输出契约，提升信任与复用。
- OA3: 以低输入成本启动研究：仅凭标题/链接就能获得结构化产出（仍需验证其来源类型与覆盖范围）。
- OA4: 把高风险事实（财务数值、政策条款）作为“强校验对象”，降低“说反/引用错”导致的灾难性错误概率（先验证其频率与真实损失）。
- OA5: 把“来源权威性 + 冲突权重裁决”显式化为输出契约的一部分，帮助用户在时间阈值内完成信任判断。
- Evidence: User statement (2026-03-16) + inference (needs validation)

## Product Hypotheses
- H1: If we can在 1–3 分钟内产出“轻量股票分析”的结构化输出 for A 股个人投资者 in 价格波动触发的买不买决策, then 用户会高频使用聊天入口, because 这是最高频触发且存在硬时间阈值。Evidence: User statement (2026-03-16)
- H2: If we can在 3–5 分钟内产出“主题深度研究”的结构化输出（结论/论点/证据/反例/催化剂） for A 股个人投资者 in 主题建仓前的系统性研究, then 研究会被完成并复用, because 超过阈值会放弃。Evidence: User statement (2026-03-16)
- H3: If we can默认提供“可核对路径”（链接/标题级）并按来源权重呈现冲突证据 for A 股个人投资者 in 快速判断可信度, then 信任与复用提升, because 用户核对方式主要是“看来源是否权威”。Evidence: User statement (2026-03-16)

## Validation Plan
- For each hypothesis: method, sample, success threshold, timeline
  - H1（轻量分析）：20 个真实提问样本；计时（≤3 分钟）、满意度（≥4/5）、是否愿意复用（≥70%）。
  - H2（深度研究）：10 个主题（AI 算力/低空经济/创新药各若干）；计时（≤5 分钟）、中途放弃（≤20%）、是否足够支撑“挑龙头”（≥70%）。
  - H3（可核对+权重）：15 个“冲突证据”样本；评测决策耗时（下降≥25%）与信任（≥4/5）。
- Data needed / queries needed:
  - 收集真实问题与主题清单（按频次排序）。
  - 定义“输出结构”固定字段（结论/论点/证据/反例/催化剂）与“情景概率”模板。
- First-priority validation choice (2026-03-16): 先验证“可核对引用”（链接/标题级），再迭代速度与冲突权重呈现。
- Output ordering preference (2026-03-16): 先给情景概率，再给证据。
- Risks (privacy, trust, error cost):
  - Trust：不可核对会快速弃用。
  - Error cost：需要明确“错一次”的代价与可接受的不确定性表达。
  - Privacy/retention：PDF“用完即删”的真实含义与技术边界需要确认（是否允许短期缓存/索引/摘要）。
