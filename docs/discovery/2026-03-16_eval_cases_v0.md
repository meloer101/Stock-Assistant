# Eval Cases v0 (Human Review, Markdown)

Scope (2026-03-16):
- Themes: AI 算力 / 低空经济 / 创新药
- Output order: 先情景概率，再证据
- Citation minimum: 仅标题（不需要链接）
- Evidence weighting for conflict/aggregation: 年报/研报/媒体=4:4:2
- First priority to validate: 可核对引用（标题级）

Expected output format (strict markdown headings; align with `theme_research_agent`):
- `## 情景概率`（包含三行：`- 乐观：` / `- 基准：` / `- 悲观：`，合计=100）
- `## 核心论点`（含 `### 乐观` / `### 基准` / `### 悲观`，每条用 `(L1)/(B1)/(S1)` 这种 ID 开头）
- `## 证据（仅标题）`（每条以 `- [年报|权重4]` / `- [研报|权重4]` / `- [媒体|权重2]` 开头，并用 `— 支撑：(ID)` 关联到论点/反例/催化剂）
- `## 反例`（每条用 `(C1)` 开头）
- `## 催化剂`（每条用 `(K1)` 开头）
- `## 核对清单`
- `## 免责声明`

How to use:
1) 把某条用例的 `Input` 丢给系统
2) 用 `Structure Checks` 做结构验收（先 Pass/Fail）
3) 用 `Rubric` 打分（0–2），记录失败点与改进建议

Global hard rules (apply to every case):
- 不写“没有出处的事实断言”（尤其财务数值、政策条款）；如果不能给标题引用，必须降级为“待核对/假设”。
- 引用只需标题，但必须能让人“去搜得到/去平台里找得到”（例如：年报章节名、研报标题、媒体报道标题）。
- 必须标注来源类型与权重：年报=4、研报=4、媒体=2（用于冲突裁决与综合打分）。

## Rubric (0–2 each)
- R1 引用可核对性：关键断言是否都给到标题引用（Pass/Fail 也可作为硬门槛）
- R2 来源类型/权重表达：是否标注年报/研报/媒体及 4:4:2，并用于冲突解释
- R3 事实/推断/情景假设区分：是否明确哪些是事实、哪些是推断、哪些是假设
- R4 结构完整度：是否按约定结构输出且信息密度足够
- R5 反例质量：反例是否能真实削弱论点且带引用
- R6 催化剂可操作性：催化剂是否可观察、可验证、带引用
