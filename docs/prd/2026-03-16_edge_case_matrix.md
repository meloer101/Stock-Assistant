# Edge Case Matrix — Vine 投资研究助手（v0.1）

| Scenario | Expected Behavior | Notes |
|---|---|---|
| 主题研究输入过于宽泛（如“帮我研究一下经济”） | 仍按固定输出结构返回，但在 `## 核对清单` 里要求用户缩小范围（行业/主题/时间窗），并将关键断言标注为待核对 | 不应编造“权威标题” |
| 情景概率不合计 100 | 视为结构性失败；系统应在生成阶段自检并修正 | 评测用例直接 Fail |
| 证据区缺少关键断言的标题引用 | 将断言降级为“待核对/假设”，并在核对清单列出需要寻找的权威标题类型 | 主题研究核心门槛 |
| 引用标题疑似编造/不在搜索结果里 | 输出中标注“未能在检索结果确认标题”，并从报告里移除该标题作为证据 | v0.1 可先靠人工抽查，后续做工具日志对齐 |
| Tavily API Key 未配置/搜索失败 | 返回明确错误原因 + 给出用户可执行的核对清单（例如“去 Wind 搜索：关键词…/去年报章节…”） | 不要沉默失败 |
| 搜索结果质量低/小媒体居多 | 在证据区降低媒体权重，强调缺少年报/研报支撑并提示风险 | 体现 4:4:2 权重逻辑 |
| 用户要求只要标题、不需要链接，但仍追问“在哪看到的” | 解释产品约束：仅标题引用；建议用户在同花顺/Wind/搜索引擎按标题检索 | 与用户偏好一致 |
| 上传 PDF 无文本（扫描件） | 返回“PDF 无可提取文本”，提示用户换可复制文本版本或使用 OCR（TBD） | 当前 parse_pdf 仅提取文本 |
| 文档问答无相关段落召回 | 明确说明“文档中未找到”，并给建议的查询改写（关键词/章节） | 保持引用规范 |
| 用户删除文档后继续提问该文档内容 | 提示文档已删除且不可再检索；建议重新上传 | 满足“用完即删”体验 |
| 用户删除会话后刷新仍可见 | 视为严重 bug；删除后前端列表与消息回放不可见 | 后端需明确物理/逻辑删除策略 |
